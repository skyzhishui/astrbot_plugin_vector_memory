# AstrBot 向量记忆插件 🧠

一个基于向量语义检索的长期记忆系统，让 AI 能够真正"记住"重要信息。

## ✨ 功能特点

- 🔍 **语义检索**: 使用向量相似度搜索，而非关键词匹配
- 💾 **自动记忆**: 自动从对话中提取重要信息并存储
- 🎯 **智能注入**: 在对话时自动检索相关记忆并注入上下文
- 📊 **分类管理**: 支持多种记忆类别（偏好、事实、事件等）
- ⭐ **重要性评分**: 每条记忆都有重要性权重
- 🚀 **ChromaDB 驱动**: 使用 ChromaDB 实现高效的向量索引和检索

## 📦 安装

1. 将插件放入 `AstrBot/data/plugins/astrbot_plugin_vector_memory/` 目录
2. 安装依赖: `pip install chromadb`
3. 在 AstrBot 管理面板中配置 Embedding Provider ID
4. 重启 AstrBot 或使用 `/memory_init` 初始化

## ⚙️ 配置说明

在 `_conf_schema.json` 中配置：

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `embedding_provider_id` | Embedding Provider ID（必填） | "" |
| `auto_remember` | 是否自动记忆对话中的重要信息 | true |
| `memory_threshold` | 自动记忆的对话轮数阈值 | 5 |
| `top_k` | 检索时返回的记忆数量 | 5 |
| `admin_ids` | 管理员ID列表 | [] |

## 🎮 使用方法

### LLM 工具（对话中使用）

直接与 AI 对话，它会自动使用这些工具：

- **记住信息**: "记住我喜欢吃草莓蛋糕"
- **回忆信息**: "你还记得我喜欢什么吗？"
- **列出记忆**: "列出你记得的所有事情"
- **删除记忆**: "忘记那条关于xxx的记忆"
- **查看统计**: "你的记忆系统状态如何？"

### 命令

- `/memory_init` - 初始化记忆系统
- `/memory_test` - 测试记忆系统是否正常工作

## 🧪 示例对话

```
用户: 记住我下周三有个会议
AI: ✓ 已记住: 下周三有个会议 (ID: 1, 类别: task)

用户: 我喜欢喝拿铁咖啡
AI: ✓ 已记住: 喜欢喝拿铁咖啡 (ID: 2, 类别: preference)

用户: 我有什么重要的事情吗？
AI: 找到 2 条相关记忆:
1. [task] 下周三有个会议
   相关度: 0.85
2. [preference] 喜欢喝拿铁咖啡
   相关度: 0.72
```

## 📁 数据存储

记忆数据存储在 `/AstrBot/data/vector_memory/` 目录（ChromaDB 持久化存储）

## 🔧 技术架构

```
用户消息 → Embedding → ChromaDB 向量搜索 → 返回相关记忆
                                    ↓
                              注入到 LLM 上下文
```

### 为什么选择 ChromaDB？

- ✅ **无需额外部署** - 嵌入式运行，像 SQLite 一样简单
- ✅ **自动向量索引** - 使用 HNSW 算法，检索速度快
- ✅ **支持元数据过滤** - 可按类别、重要性等条件筛选
- ✅ **Python 原生** - API 简洁，易于集成

## 📝 记忆类别

| 类别 | 说明 | 示例 |
|------|------|------|
| `preference` | 用户偏好 | 喜欢吃草莓蛋糕 |
| `fact` | 重要事实 | 生日是3月15日 |
| `personal` | 个人信息 | 工作是程序员 |
| `event` | 重要事件 | 下周三有会议 |
| `task` | 任务承诺 | 答应帮忙写代码 |
| `general` | 通用记忆 | 其他信息 |
